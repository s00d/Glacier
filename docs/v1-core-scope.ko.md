# Glacier v1 핵심기능 범위

작성일: 2026-03-13

## 문서 역할

- 이 문서는 Glacier의 `v1에서 반드시 해내야 하는 핵심기능`을 고정하는 문서다.
- PRD의 방향을 실제 제품 범위로 내리는 것이 목적이다.
- 구현 세부 구조는 `refactor-architecture.ko.md`에서 다룬다.

## v1 제품 한 줄 정의

Glacier v1은 `정리된 메뉴바를 유지하면서, 숨긴 메뉴바 아이템을 필요할 때 바로 꺼내 실제로 사용할 수 있게 하는 최소한의 메뉴바 도구`여야 한다.

## v1 핵심 성공 조건

다음 네 가지가 되면 v1의 본질은 성립한다.

1. 사용자가 메뉴바를 정리된 상태로 유지할 수 있다.
2. 사용자가 숨긴 아이콘을 한 번에 꺼낼 수 있다.
3. 펼친 뒤 실제로 그 아이콘을 클릭해 사용할 수 있다.
4. 문제가 생겨도 사용자가 스스로 원상복구할 수 있다.

## v1 핵심기능

### 1. 숨김 영역 펼치기 / 접기

설명:

- 사용자는 메인 컨트롤을 눌러 hidden 영역을 펼칠 수 있어야 한다.
- 다시 누르거나 명확한 종료 행동으로 접을 수 있어야 한다.

v1 판단:

- 필수

### 2. 펼친 아이콘의 실제 사용 가능

설명:

- 펼쳐진 hidden 영역의 아이콘을 클릭해 실제 메뉴/팝오버/액션을 사용해야 한다.
- Glacier가 먼저 접혀 사용을 방해하면 안 된다.

v1 판단:

- 최우선 필수

### 3. 상태 구분

설명:

- 닫힘
- hidden 펼침
- all 펼침

세 상태가 최소한 시각적으로 구분되어야 한다.

v1 판단:

- 필수

### 4. 사용법 진입점

설명:

- 우클릭 또는 명시적 진입점에서 사용법을 다시 볼 수 있어야 한다.

v1 판단:

- 필수

### 5. 편집 진입점

설명:

- 사용자는 "어떻게 구역을 다시 조정하는지" 찾을 수 있어야 한다.
- 편집은 일상 사용과 분리된 진입점이 있어야 한다.

v1 판단:

- 필수

### 6. 레이아웃 복구

설명:

- marker 위치가 꼬이거나 구성이 헷갈릴 때 기본 상태로 초기화할 수 있어야 한다.

v1 판단:

- 필수

## v1 기능 정책

### 보임 / 숨김이 기본 개념이다

v1에서 사용자가 가장 먼저 이해해야 하는 개념은:

- 항상 보임
- 숨김

두 가지다.

### always hidden은 고급 기능으로 둔다

`always hidden`은 남긴다. 하지만 v1의 첫인상과 기본 안내에서는 전면에 두지 않는다.

이유:

- 초반 인지 부하를 크게 늘린다.
- 대부분 사용자는 `보임 / 숨김`만으로도 충분하다.
- 고급 사용자는 이후에 확장 개념으로 받아들일 수 있다.

## v1에서 하지 않을 것

- 복잡한 설정창
- 세밀한 자동화 규칙
- 항목별 정책 편집 UI
- 광범위한 시각 커스터마이즈
- 사용자에게 여러 조작 제스처를 외우게 하는 설계

## 우선순위

### P0

- hidden 펼치기 / 접기
- 펼친 아이콘 실제 사용 가능
- 상태 시각 구분
- 사용법 진입점

### P1

- 편집 모드 진입
- 레이아웃 초기화
- 첫 실행 안내 개선

### P2

- always hidden 고급 안내
- 세부 타이머 정책
- 시각 표현 다듬기

## 출시 판단 기준

다음 질문에 모두 "예"라고 답할 수 있어야 v1로 간주한다.

- 메뉴바를 정리된 상태로 유지할 수 있는가
- 숨긴 아이콘을 다시 꺼내는 과정이 빠른가
- 꺼낸 뒤 실제 사용이 가능한가
- 처음 쓰는 사람도 구조를 다시 찾을 수 있는가
- 꼬였을 때 reset 가능한가

## 최종 정리

v1의 핵심은 기능 수가 아니라 행동 성공률이다.

즉, Glacier v1은 `많이 숨기는 앱`이 아니라 `숨긴 뒤에도 확실히 다시 쓸 수 있는 앱`이어야 한다.
