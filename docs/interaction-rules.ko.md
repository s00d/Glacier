# Glacier 행동 규칙 명세

작성일: 2026-03-13

## 문서 역할

- 이 문서는 Glacier가 어떤 입력에 어떻게 반응해야 하는지 명확히 정의한다.
- 목표는 "펼친 뒤 실제 사용 가능"을 보장하는 것이다.

## 기본 원칙

1. 펼침은 사용 진입이다.
2. 접힘은 사용 종료 뒤에 일어난다.
3. 사용자가 예상하는 행동을 방해하지 않는다.

## 컨트롤 정의

- `●`: 기본 컨트롤
- `◆`: 고급 경계 마커

## 입력 규칙

### 1. `●` 좌클릭

현재 상태가 닫힘이면:

- hidden 영역을 펼친다.

현재 상태가 hidden 펼침이면:

- 접는다.

현재 상태가 all 펼침이면:

- 접는다.

### 2. `Option + ●` 클릭

현재 상태가 닫힘이면:

- all 펼침 상태로 전환한다.

현재 상태가 all 펼침이면:

- 접는다.

현재 상태가 hidden 펼침이면:

- all 펼침으로 확장한다.

### 3. `◆` 좌클릭

현재 상태가 hidden 펼침이면:

- all 펼침으로 확장한다.

현재 상태가 all 펼침이면:

- hidden 펼침으로 돌아간다.

### 4. `●` 우클릭

- 컨텍스트 메뉴를 연다.
- 최소 항목:
  - 사용법
  - 편집 모드
  - 레이아웃 초기화
  - 종료

## 접힘 규칙

### 접어도 되는 경우

- 사용자가 `●`를 다시 클릭함
- 사용자가 `Esc`를 누름
- 메뉴바 바깥의 빈 공간을 클릭함
- 일정 시간 동안 상호작용이 완전히 끝난 뒤 자동 정리 타이머가 만료됨

### 접으면 안 되는 경우

- 펼쳐진 hidden 영역의 아이콘을 클릭함
- 펼쳐진 아이콘이 연 메뉴와 상호작용함
- 펼쳐진 아이콘이 띄운 팝오버와 상호작용함
- 사용자가 편집 모드에서 marker를 재배치 중임

## 자동 접힘 정책

v1에서는 자동 접힘보다 사용 성공이 우선이다.

따라서 자동 접힘은 다음처럼 보수적으로 둔다.

- 외부 클릭 즉시 접기: 허용
- 단, 펼쳐진 메뉴바 아이콘 사용 클릭은 외부 클릭으로 취급하지 않음
- 무활동 타이머: 선택적
- 메뉴/팝오버 사용 중 자동 접힘: 금지

## 편집 모드 규칙

편집 모드에 들어가면:

- marker 의미가 더 분명하게 보여야 한다.
- 사용 모드보다 편집 동작이 우선된다.
- 실수 방지를 위해 reset 진입점이 보여야 한다.

편집 모드에서는:

- 드래그 조정 허용
- 접힘 정책 완화
- 상태 안내 강화

## 상태 표시 규칙

### 닫힘

- 최소 표현
- 정리된 메뉴바 상태

### hidden 펼침

- 사용 가능한 상태라는 점이 보여야 함
- `●`의 시각 표현이 닫힘과 달라야 함

### all 펼침

- hidden 펼침보다 더 강한 강조가 필요함
- 고급 상태라는 점이 드러나야 함

### 편집 모드

- 현재 사용 모드가 아니라는 점이 즉시 보이게 해야 함

## 금지 패턴

- 펼친 직후 다른 아이콘을 클릭하자마자 Glacier가 먼저 접히는 것
- 클릭과 동시에 상태가 예측 불가능하게 바뀌는 것
- 사용 모드와 편집 모드가 같은 제스처 체계로 뒤섞이는 것
- reset 없이 배치 실패 상태가 지속되는 것

## 예시 시나리오

### 시나리오 A: 일반 사용

1. 사용자가 `●` 클릭
2. hidden 영역 펼침
3. 사용자가 VPN 아이콘 클릭
4. VPN 메뉴 사용
5. 메뉴 종료
6. 다시 접기 또는 유예 후 접기

### 시나리오 B: 고급 사용

1. 사용자가 `Option + ●` 클릭
2. all 펼침
3. rarely used 아이콘 접근
4. 작업 완료
5. 접기

### 시나리오 C: 편집

1. 사용자가 우클릭 메뉴에서 편집 모드 진입
2. marker/구역 의미 확인
3. 재배치
4. 완료 또는 reset
5. 기본 상태 복귀

## 최종 정리

Glacier의 핵심 행동 규칙은 단순하다.

- 펼치면 쓸 수 있어야 한다.
- 쓰는 동안 방해하면 안 된다.
- 끝났을 때만 정리해야 한다.
