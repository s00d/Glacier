# Glacier 복구 경로 및 온보딩 설계

작성일: 2026-03-13

## 문서 역할

- 이 문서는 사용자가 길을 잃지 않도록 만드는 복구 장치와 첫 사용 안내를 정의한다.
- 목적은 "작은 앱이지만 스스로 회복 가능한 제품"으로 만드는 것이다.

## 왜 필요한가

Glacier는 메뉴바 배치를 직접 다룬다. 이런 앱은 기능이 적더라도 다음 문제가 쉽게 생긴다.

- marker 의미를 잊음
- 구역 배치를 잘못함
- 펼쳤는데 왜 다시 접히는지 이해하지 못함
- 현재 상태를 파악하지 못함

따라서 복구 경로와 온보딩은 부가 기능이 아니라 핵심 기능이다.

## 필수 복구 경로

### 1. 사용법 다시 보기

위치:

- 우클릭 메뉴

역할:

- `●`, `◆`, 구역 의미, 기본 조작을 다시 설명

### 2. 레이아웃 초기화

위치:

- 우클릭 메뉴
- 편집 모드 내부

역할:

- marker 위치와 상태를 기본값으로 복구

### 3. 안전 종료

위치:

- 우클릭 메뉴

역할:

- 현재 상태와 무관하게 앱을 안정적으로 종료

### 4. 편집 모드 재진입

위치:

- 우클릭 메뉴

역할:

- 사용자가 "다시 정리해야겠다"는 판단을 했을 때 바로 진입 가능

## 첫 실행 온보딩

### 목표

- 5초 안에 앱의 핵심 개념을 이해시킨다.

### 전달해야 하는 내용

1. `●`는 숨긴 항목을 여는 컨트롤이다.
2. 펼친 뒤 실제로 아이콘을 클릭해 사용할 수 있다.
3. 편집/복구는 우클릭 메뉴에서 다시 들어올 수 있다.

### 피해야 하는 것

- 너무 긴 설명
- 3구역 전체를 한 번에 과도하게 설명
- 여러 단축키 암기 요구

## 권장 구조

### 첫 실행 안내

- 짧은 3단계 안내
- 한 번 닫아도 우클릭 메뉴에서 다시 확인 가능

### 우클릭 메뉴

- 사용법
- 편집 모드
- 레이아웃 초기화
- 종료

### 편집 모드 보조 안내

- 구역 라벨
- 완료
- reset

## 오류 상황별 복구

### 상황 A: marker 의미를 잊음

복구:

- 사용법 열기

### 상황 B: 배치를 잘못 옮김

복구:

- 편집 모드 진입
- reset 또는 다시 배치

### 상황 C: 현재 열린 상태가 헷갈림

복구:

- 상태 표시 강화
- `Esc`로 안전하게 닫기

### 상황 D: 사용 도중 동작이 이상함

복구:

- 모두 닫기
- reset
- 다시 시작

## 운영 원칙

- 사용법은 언제든 재진입 가능해야 한다.
- reset은 숨기지 않는다.
- 종료는 항상 마지막 안전장치로 남긴다.
- 편집은 사용과 분리하되 접근은 쉽다.

## 최종 정리

Glacier는 작은 앱이지만 작은 앱일수록 복구 경로가 중요하다.

사용자가 길을 잃었을 때 다음 세 가지가 항상 가능해야 한다.

- 도움말 다시 보기
- 기본 상태로 되돌리기
- 편집으로 다시 들어가기
